# Documentation Rules

## Structure
- All project documentation lives in `documentation/`
- Each subfolder should have a `README.md` index describing its contents
- When a user references a requirement or design spec, search `documentation/` first

## Before Implementing
- Read the relevant spec in `documentation/` before writing code
- If no spec exists for the requested feature, note this to the user

## Keeping Docs Current
- When implementation diverges from a spec, flag it to the user
- Do not modify documentation files unless explicitly asked
